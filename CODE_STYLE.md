# 项目C语言代码规范

## 1. 缩进和格式

- 使用 4 空格进行缩进，不要使用制表符
- 大括号 `{}` 必须单独占一行, 不要将`{`放在行尾
- 条件比较时，常量在前, 如 `NULL == ptr` 而不是 `ptr == NULL`
- `while`、`if`、`for` 等关键字与左括号 `(` 之间需要有一个空格, 例如: `if (1 == 1)`
- 不允许省略`{}`, 例如:
```
    /* 正确的写法 */
    if (true == a)
    {
        printf("true\n");
    }

    /* 错误的写法 */
    if (false == a)
        printf("false\n");
```
- 表达式符号前后需要有空格, 例如: `1 + 1` `1 & 1` `1 | 1`, 但自增自减的表达式不应有空格, 例如: `a++`
- 复杂表达式需要添加`()`, 像`a * b + 1`即使`*`运算优先度高, 也应写成`(a * b) + 1`
- 所有类型的变量在定义时必须有初始化, 如`int num = 0`, `char* ptr = NULL`, `int numvector[10] = {0}`
- 变量定义时，函数名需要对齐
- 变量定义完成后的使用、赋值等操作时, 等号 `=` 无需对齐
- 类型和变量名之间只需要一个空格
- 指针变量的星号 `*` 紧跟类型如 `char* buffer` 而不是 `char *buffer`
- 在一行中, 如果超长，则以参数对齐, 一个参数一行

## 2. 结构体和枚举
**结构体**
- 结构体定义格式：`typedef struct 大写 { 结构体成员 } 小写;`
- 结构体内的成员变量根据名称对齐，注释也要对齐
- 结构体名称允许带有`_`, 例如: EXAMPLE_STRUCT, 但结构体内的成员变量名称需要是具有意义的名称, 要以`m_`开头且不允许带有其他`_`, 例如`m_param1`, 而不是`m_param_1`  
举例:
```
typedef struct EXAMPLE_STRUCT
{
    bool m_success;
    int m_type;
    char* m_name;
    char m_buff[64];
} example_struct;
```

**枚举**
- 枚举定义格式: `typedef enum 大写枚举名 {大写枚举名_大写值名} 小写枚举名`
- 枚举的成员赋值时只允许对第一个成员赋值, 不允许每个成员均赋值  
- 枚举成员后的`,`要对齐
举例:
```
typedef struct EXAMPLE_ENUM
{
    EXAMPLE_ENUM_INIT = 0x01,
    EXAMPLE_ENUM_RUNNING,
    EXAMPLE_ENUM_STOPED
} example_enum;
```

## 3. 函数
- 函数名称格式：`模块名称_功能()`
- 函数内局部变量统一放到函数头部声明, 不允许混合声明, 也不允许在函数内部的if, while, for等内部声明变量
- 函数内变量声明时按照占用字节顺序由小到大定义, 如：`bool` → `int` → `long` → `pointer`  
代码示例:
```
bool failure = false;
int num      = 0;
long numlong = 0;
char* ptr    = NULL;
```

## 4. 头文件
- 头文件中不能包含其他头文件, 总控头文件除外
- 头文件必须使用 `#ifndef` 防止重复包含
- 头文件最后有且仅有一行为空行

## 5. 源文件
- 源文件要以`项目名称缩写_模块名称.c`格式命名, 若模块内包含多个子文件, 那么以`项目名称缩写_模块名称_子模块名称.c`格式命名
- 源文件内的全局变量需要以`g_`开头, 文件内static变量需要以`m_`开头
- 头文件最后有且仅有一行为空行
