# pgcastor-pro Project Analysis

## Project Overview

**pgcastor-pro** (also referred to as **XSynch**) is a PostgreSQL data replication and synchronization toolkit written in C. It provides a comprehensive solution for capturing, transforming, and applying database changes between PostgreSQL instances.

### Purpose
- Real-time data capture from PostgreSQL WAL (Write-Ahead Log)
- Data transformation and trail file generation
- Data integration/application to target databases
- Centralized management through a CLI tool and manager process

### Technology Stack
| Component | Technology |
|-----------|------------|
| Language | C (C99 standard) |
| Build System | CMake 3.10+ |
| Parser Generator | Bison + Flex |
| Database | PostgreSQL 12+ |
| Compression | LZ4 |
| CLI Library | Readline |

## Executable Components

### 1. capture
**Location:** `bin/capture/capture.c`

**Purpose:** Captures data changes from PostgreSQL source database.

**Functionality:**
- Connects to source PostgreSQL database
- Reads and parses WAL (Write-Ahead Log) files
- Extracts DML/DDL operations (INSERT, UPDATE, DELETE, CREATE, ALTER, etc.)
- Generates trail files containing captured changes
- Supports table filtering (include/exclude patterns)
- Handles big transactions specially
- Supports online refresh for initial data synchronization

**Operations:** `init`, `start`, `stop`, `status`, `reload`, `onlinerefresh`

**Configuration:** `etc/capture.cfg`

### 2. integrate
**Location:** `bin/integrate/integrate.c`

**Purpose:** Applies captured changes to target PostgreSQL database.

**Functionality:**
- Reads trail files generated by capture
- Parses and applies DML/DDL operations to target database
- Manages transaction consistency
- Handles conflict resolution
- Supports parallel processing with multiple worker threads

**Operations:** `init`, `start`, `stop`, `status`, `reload`

**Configuration:** `etc/integrate.cfg`

### 3. xmanager
**Location:** `bin/xmanager/xmanager.c`

**Purpose:** Central manager process for coordinating all XSynch components.

**Functionality:**
- Acts as a central hub for all XSynch jobs
- Provides network services (TCP and Unix Domain Socket)
- Monitors and manages capture/integrate processes
- Handles heartbeat and health checks
- Provides metrics and status information

**Operations:** `init`, `start`, `stop`, `status`, `reload`

**Configuration:** `etc/xmanager.cfg`

**Network:**
- Default port: 6543
- Supports TCP and Unix Domain Socket connections

### 4. receivepglog
**Location:** `bin/receivepglog/`

**Purpose:** Receives PostgreSQL WAL logs (pg_receivewal equivalent).

**Functionality:**
- Streams WAL files from PostgreSQL server
- Provides backup/HA capabilities for WAL logs

**Configuration:** `etc/receivewal.cfg`

### 5. xscsci
**Location:** `bin/xscsci/xscsci.c`

**Purpose:** Command-line interface tool for managing XSynch jobs.

**Functionality:**
- Interactive shell with readline support
- Connects to xmanager for job management
- Supports job lifecycle commands:
  - `create` - Create new jobs (manager, capture, integrate, pgreceivelog)
  - `edit` - Edit job configuration files
  - `init` - Initialize job working directories
  - `start` - Start jobs
  - `stop` - Stop jobs
  - `reload` - Reload configuration
  - `remove` - Remove configuration files
  - `drop` - Drop jobs
  - `info` - View job information
  - `watch` - Monitor job status in real-time
  - `alter` - Modify progress members

**Parser:** Uses Bison/Flex for command parsing (`xscsci_gram.y`, `xscsci_scan.l`)

## Data Flow

```
1. CAPTURE PHASE
   PostgreSQL ──WAL──► capture ──Trail Files──► Storage

2. INTEGRATE PHASE
   Storage ──Trail Files──► integrate ──SQL──► Target PostgreSQL

3. MANAGEMENT PHASE
   xscsci (CLI) ──Commands──► xmanager ──Control──► capture/integrate
```

### Trail Files
- Binary format containing captured changes
- Configurable size (default: 16MB)
- LZ4 compression support
- Located in capture's `data` directory
